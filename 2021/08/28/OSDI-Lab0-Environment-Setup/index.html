2021-08-28

Cross-platform development

Cross compiler

當編譯工具使用的指令集和編譯目標使用的指令集不同時，就需要使用 cross-compiler 進行編譯。rpi4 使用 ARM 指令集，因此需要使用 cross-compiler [1]。

required Install the cross-compiler on your host computer.

question What’s the RAM size of rpi4?

Memory : 4G

question What’s the cache size and level of rpi4?

Caches : 32 KB data + 48 KB instruction L1 cache per core. 1MB L2 cache.

Linker

linker script 是提供給 linker 參考的文件，它告訴 linker 我想要怎麼擺放這些 section，甚至也可以定義程式的起始點在哪邊。

question Explain each line of the above linker script.

link 完成之後的 binary text section 要放在位址 0x80000，假設這個 binary file 的材料有 abcd 四個 object file 都有 text section，就要依照某種神秘順序從 0x80000 開始往下放。

*(.text) 表示所有 object file 的 text section 都要移動，如果使用 a.o(.text) 就只有檔案 a 會搬到位址 0x80000。

From source code to kernel image

From source code to object files

a.S 是我們要使用的 source code，指令 wfe [2] 是進入低耗能狀態，指令 b 是 jump，因此這段 code 是在執行一個無窮迴圈。

使用 cross compiler 把 source code 編譯成 object file，a.S 是 soucre code，使用 -c 表示編成 object file，編譯出來的 object file 是 a.o。

From object files to ELF

ELF [3] 是一種檔案格式，各種 UNIX 系統的可執行檔都採用 ELF 格式，平常用 gcc 編譯出來的檔案就是 ELF 格式。我們使用助教善心提供的 linker script 把 object file link 成 ELF file，-T linker.ld 表示使用 linker.ld 這個 linker script。

我們可以使用 gnu-objdump 來看一下 kernel8.elf 裡面有什麼，-D 表示要顯示出所有 sections 的內容。

因為 a.S 只有 text section，所以我們的 ELF 裡面也只有 text section，並且依照 linker script 的指示，text section 被放在位址 0x80000，依序是第一道指令 wfe 和第二道指令 b，並且指令 b 要跳到 _start，也就是位址 0x80000。

From elf to kernel image

雖然 ELF 已經是平常使用的可執行檔案格式了，但因為今天我們要把他做成 kernel image，所以必須把 ELF 轉換成 binary 格式。ELF 格式包含許多 meta data，binary 格式則是純 binary file，我猜測或許是因為 ELF 的 metadata 是給 OS 看的，但在安裝 OS kernel 時還沒有 OS 所以不能有 metadata 在其中因為沒有人看的懂。 gnu 指令 objcopy 可以複製或是轉換，怎麼轉換要看他立的 flag，這裡下 -O 表示要轉換成 binary。

Deploy to rpi4

為了可以用 SD card 開機 rpi4，我們需要對 SD card 做一些準備，當我們將 SD card 連接到電腦，可以看到在資料夾 /dev 多了兩個裝置 sda 和 sda1，因為 /dev 是 linux 管理所有 device 的地方。

sda 是這張 SD card， sda1 是 SD card 裡面的 partition，我們要做格式化是要對 partition 做格式化，因為每個 partition 可以也不同的格式。我們使用指令 mkfs 來進行格式化，但得到錯誤訊息，錯誤訊息表示這個 SD card 不能在 mount 的狀態進行格式化，因此我們在 umount SD card 之後再重新格式化一次，即可成功。

要修改 partition type，因為 rpi4 在開機的時候會使用到這個資訊，修改的時候要對 SD card 而不是 partition 做修改，因為 partition type 是 SD card 的 metadata。

完成 partition 之後要準備 booting firmware，我們準備了 fixup4.dat 和 start4.elf，rpi4 不需要 bootcode.bin 因為已經燒在板子上了。把這兩份 code 和上面編譯好的 kernel8.img 一起放進 partition，SD card 的準備就完成了。

Interact with rpi4

要讓 rpi4 動起來，要把以下幾個檔案放進去，除了 kernel8.img 要自己編譯之外，另外三個都可以在 [4] 下載。

  • kernel8.img : kernel image
  • start4.elf : 簡單的 firmware [5]
  • fixup4.dat : 和 start*.elf 搭配的 linker files
  • bcm2711-rpi-4-b.dtb : 硬體相關資料

使用 UART [6] 連接電腦和 rpi4，需要將 TX (transport) / RX (receive) / GND 連接到 rpi4 的相應的 pin，黑色線接 GND (上排左三)，白色線接 TX (上排左四)，綠色線接 RX (上排左五)。

完成連接之後，使用 screen 和 rpi4 溝通，ttyUSB0 是 rpi4，下完這個指令之後，將 rpi4 接電，rpi4 就會開機，但是這個時候我們還沒有讓 rpi4 開機之後印出任何東西，所以雖然他有開機，但我們看不出來，等到下一個 lab 實做 shell 的時候，就可以看到 rpi4 和我們溝通了。
